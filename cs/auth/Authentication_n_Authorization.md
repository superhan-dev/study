# Authentication / Authorization

인증과 인가의 차이를 알고 시작하는 것이 중요하다.
인증은 접근을 허가 받는 것을 의미한다. 사용자가 접근을 시도했을 떄 입장 가능한 사용자인지 입장권을 확인하는 작업이라고 할 수 있다.
인가는 특정 리소스에 접근권한이 있는지 확인하고 허가하는 과정이다.

입장권을 받고 입장했지만 손댈 수 없는곳이 정해져 있고 접근을 시도하면 차단하는 절차이다.

그렇다면 엑세스 토큰을 사용하는 것인 인증인가요? 인가인가요?

> 엑세스 토큰을 사용하는 것은 인가라고 할 수 있다.

## 실제 사용

서비스에서 인증은 회원 로그인을 의미하고 API 요청을 통해 값을 가져올때 Access Key를 사용하는 것은 인가 과정일 것이다.
이때 주로 사용하는 토큰이 JWT 토큰이다.

### JWT란?

json web token을 의미하며 사용자 속성 정보를 저장하는 Claim 기반의 웹토큰이다.
토큰 자체를 정보로 사용하는 Self-Contained 방식으로 정보를 안전하게 전달한다.
토큰 생성시에 특정 Key값을 사용해 발행을 하므로 Key값을 모르는 외부에서는 토큰을 위조할 수도 없고 복호화 할 수도 없다.
만약 Key가 노출된다면 가능한 시나리오이긴 하지만 의도적인 노출이 아니라면 크게 걱정할 필요는 없다.

### JWT 구조

JWT는 일종의 규격이다. Header, Payload, Signature로 구성된다. 또한 JSON 형태인 각 부분은 Base64로 암호화 된다.
