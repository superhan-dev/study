# 인스턴스화를 막으려면 private 생성자를 사용하라.

우선 이말이 무슨 말인지 정리해보자.

## private 사용자를 사용한 인스턴스화 방어

다음과 같이 정의하면 전역에서 Config객체가 인스턴스화 되는 것을 막을 수 있다.

```java
public class Config {
    private static final Config INSTANCE = new Config();

    private Config(){...} // 외부에서 new를 사용하여 생성하지 못하게 막는다.

    public static Config getInstance(){
        return INSTANCE;
    }
}
```

## 왜 사용해야 할까?

생성하는 방법을 통제하기위해 사용한다. 생성 디자인 패턴을 사용할 때 펙토리 메서드, 빌더, 싱글턴 등 인스턴스 생성을 제어해야하는 상황이 발생한다. 이럴때 주로 private 키워드를 사용해서 전역 인스턴스화를 막는다.
