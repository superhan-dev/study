# 1부

## 소프트 웨어란 무엇인가?

소프트웨어를 구현하는데 있어 변경이 용이하고 요구사항에 유연하게 대응할 수 있는 시스템을 구축하는 것이 매우 중유하다.

이러한 이유가 아키텍처라는 용어가 프로그램에서 사용되는 이유일 것이다.
옳고 그른 아키텍처가 있는 것은 아니다. 용도에 따라 사용될 뿐이다.

> 변경에 유연하지 않다면 그것은 소프트웨어가 아니라 하드웨어다.

# 2부

# 3가지 페러다임

- 구조적 프로그래밍
- 객체 지향 프로그래밍
  - 힙에 인스턴스가 존재하는 이유는?
    - 알골 언어의 함수호출 스텍 프레임을 힙으로 옯기면, 함수 호출이 반환된 이후에도 함수에서 선언된 지역변수가 오랫동안 유지될 수 있음을 발견했다. 이러한 함수가 클래스의 생성자가 되었고, 지역변수는 인스턴스 변수, 그리고 중첩함수는 메서드가 되었다. 함수 포인터를 특정 규칙에 따라 사용하는 과정을 통해 필연적으로 다형성이 등장했다.
- 함수형 프로그래밍
  - 람다 계산법이 발명되었고 함수형 프로그래밍도 이 계산법에 직접적인 영향을 받아 발전하기 시작했다. 함수형 언어에서 사용되는 람다의 기초 개념은 불변성이다. 심볼의 값이 변경되지 않는다는 의미의 개념이다.

## 구조적 프로그래밍

goto와 같이 특정 위치로 jump를 제공하지 않기 때문에 오로지 조건문과 반복문이 현대의 프로그래머들이 대표적으로 사용하는 구조적 프로그래밍 기법이다.

## 객체지향 프로그래밍

### Object Oriented 의 4가지 특징은 캡상추다.

- 캡슐화
  - 메소드나 지역 변수가 외부에서 접근할 수 있는 기능을 제공하지 않음으로써 값이 변경되는 영역을 객체 안으로 제한함으로써 안전하고 신뢰할 수 있는 로직을 구현할 수 있도록 제공한 기능이다.
  - 하지만 OO 언어라고 해서 모든언어가 캡슐화를 완벽하게 제공하는 것은 아니다.
- 상속
  - 하위 객체에서 상위 객체의 속성과 메소드를 그대로 이어받아 확장 또는 재사용의 개념으로 사용되는 특징을 상속이라고 한다.
- 추상화
- 다형성
