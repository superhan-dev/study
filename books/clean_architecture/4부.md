# 컴포넌트 원칙

# 12장 컴포넌트

컴포넌트란 하나의 배포단위를 의마하며 jar, war, dll등과 같은 단위로 프로그램을 배포함을 의미한다.

초기 프로그램은 컴파일 이후 메모리에 해당 함수를 배치해서 동작시키는 방식으로 동작했고 프로그램이 커질수록 할당 가능한 공간을 넘어서게 되었다.
때문에 동작해야하는 함수만 메모리 위에서 동작시키는 것이 필요했다.

## 재배치성

해결책으로 재배치가 가능한 바이너리 형태의 프로그램을 컴파일해서 지능적인 로더를 사용해 메모리에 바이너리를 로드하는 방식으로 프로그램을 구현하기 시작했다.

이후 컴파일러가 바이너리 안의 함수 이름을 메타데이터로 생성하도록 수정되었고 외부 참조를 외부 정의에 링크시킬 수 있는 링크 로더가 탄생하게 되었다.

## 링커

링킹 로더의 등장으로 작은 라이브러리들은 해결이 되었으나 프로그램이 점점 커지면서 링킹 로더도 느려서 사용이 불가할 지경에 다다르기 시작했다.

이를 해결하기 위해 링커라는 별도의 어플리케이션으로 링크가 완료된 재배치 코드를 만들어 사용할 수 있도록 했다. 실행파일을 만드는 과정은 길었지만 이미 만들어진 실행파일은 언제라도 빠르게 로드할 수 있었다.

## 향상된 메모리

이후 메모리 성능이 점점 향상되면서 문제점들은 점차적으로 수그러들었다.
램의 크기가 커지고 비용이 줄어들면서 링크 시간이 초단위 수준이 될 정도로 감소했고 .jar파일과 같은 공유 라이브러리들이 등장하기 시작했다.

## REP(Reuse/Release Equivalence Principle)

재사용/릴리스 등가 원칙

재사용 단위는 릴리즈 단위와 같다.
